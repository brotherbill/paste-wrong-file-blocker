# Contributing to Paste Wrong File Blocker

Paste Wrong File Blocker is a safety‑critical subsystem.  
All contributions must comply with MY_RULES.md.

## Requirements

### 1. Doctrine Compliance
Every change must align with:
- Determinism doctrine  
- Audience doctrine  
- UB doctrine  
- Identity wrapper rules  
- Override protocol  
- Change control doctrine  

If a change conflicts with MY_RULES.md, the change is invalid.

### 2. Workflow

1. Create a branch  
2. Make changes  
3. Ensure all files include correct identity wrappers  
4. Ensure no drift from MY_RULES.md  
5. Submit a pull request  
6. Document rationale in the PR description  

### 3. Prohibited Contributions
- Ambiguous behavior  
- Hidden defaults  
- Undocumented changes  
- Modifying MY_RULES.md without following change control protocol  
- Adding new file types without defining rules  

### 4. Review Expectations
Reviewers verify:
- Determinism  
- Reproducibility  
- Clarity  
- Alignment with doctrine  
- Correct identity wrappers  
- No UB introduced  

### 5. Communication
All discussions must be:
- dry  
- literal  
- procedural  
- operator‑grade  

No metaphors, humor, or narrative drift.
